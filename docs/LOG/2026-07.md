  logs) vs 4 shipped commits:

  - The chronicler wrote its log entry AFTER the session commit, so every
    session ended with a dirty tree; the next session reverted the entry.
    The #4 session log was destroyed this way, and committed log content

    commit.

    prefill ~140 tok/s, decode ~10–14 tok/s. Gemma 4's built-in MTP
    speculation kept disabling itself because verification under memory
    pressure cost 176ms vs 69ms plain decode.
- Fixes applied:
  - agent-loop.sh: lock staleness is now PID-liveness (never clear a live
    session's lock); hard per-trigger timeout GOXSD_SESSION_TIMEOUT
    (default 7200s); a failed trigger no longer aborts the sequence.
  - foreman/develop/WORKFLOW: dirty trees are RESCUED via
    `git stash push -u -m "rescue <ts>"`, never cleaned/reverted;
    two-rejection hard cap enforced via counted todos; chronicler log is
    written BEFORE the final commit and travels in it.
  - mason: edit rejected files instead of rewriting them; spec-derived
    data tables must come from deterministic generators (D1), never
    hand-typed.
  - opencode.json: model → gemma4:31b-mlx-64k (num_ctx 65536 baked into a
    derived Ollama model, verified to override the server env),
    small_model → gemma4:e4b-mlx-32k; context limits declared so opencode
    compacts before overflow.
- Next: issue filed to replace hand-written builtin/gen_types.go with a
  real generator (issue #2's byte-identical-regeneration acceptance was
  not actually met); #3 must be generator-first.
