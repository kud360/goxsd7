# hfnextract (milestone M1 — not yet implemented)

Extracts the normative builtin-datatype definitions from
`docs/specs/md/xmlschema11-2.md` into checked-in Go tables under
`builtin/`, so builtins are bootstrapped from the spec rather than
hand-transcribed:

- **Appendix E function definitions** ("hfn"): anchor `id="ap-funcDefs"`,
  individual functions `id="f-*"` (e.g. `f-decimalLexmap`,
  `f-booleanCanmap`), per-type mapping sections
  `id="<type>-lexical-mapping"` / `id="<type>-canonical-mapping"`.
- **Per-type property tables** (§3): ordered / bounded / cardinality /
  numeric, applicable facets (`cos-applicable-facets`), primitive
  ancestry, whiteSpace defaults.

Output requirements: generated files carry a `// Code generated by
hfnextract. DO NOT EDIT.` header, are deterministic (byte-identical on
regeneration — STYLE D1), and cover all 49 builtin types (acceptance
criterion in docs/PLAN.md M1).
